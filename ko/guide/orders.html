주문 개요 ​

VpAdmin의 주문 페이지는 웹사이트와 관련된 결제 주문을 관리하기 위한 포괄적인 인터페이스를 제공합니다. 여기에서 주문 상세 정보를 보고, 결제 상태를 추적하고, 전송 실패 시 이메일을 수동으로 재전송할 수 있습니다. 이 모듈은 귀하의 판매 및 구독을 효율적으로 관리하고 모니터링하는 데 도움이 됩니다.

개요 ​

주문 페이지에서는 다음을 수행할 수 있습니다.

  • 주문 목록 보기: 프로젝트 이름, 주문 번호, 생성 시간, 금액, 고객 이메일, 이메일 전송 상태와 같은 주요 정보가 포함된 모든 주문을 볼 수 있습니다.
  • 주문 필터링: 프로젝트 이름, 주문 번호, 생성 시간 및 고객 이메일을 기준으로 주문을 필터링할 수 있습니다.
  • 주문 상세 정보 보기: 원시 데이터 및 관련 설정을 포함하여 각 주문에 대한 전체 세부 정보를 볼 수 있습니다.
  • 이메일 재전송: 초기 전송에 실패한 경우 주문 확인 이메일을 수동으로 재전송할 수 있습니다.

Ko-fi 웹훅 URL 설정 ​

VpAdmin 내에서 웹사이트 주문을 관리하려면 웹훅 URL을 설정해야 합니다. Ko-fi 계정에서 웹훅 URL을 다음 주소로 구성하십시오. VpadminClientID를 VpAdmin 내 사이트 세부 정보 보기에서 찾을 수 있는 클라이언트 ID로 바꾸십시오.

웹훅 URL

https://api.vpadmin.org/prod-api/vpadmin/orders/callback/VpadminClientID

참고

관리자 기능이 활성화된 웹사이트만 클라이언트 ID를 갖습니다.

Ko-fi 웹훅 설정

주문 관리 및 자동/수동 응답의 원리에 대한 자세한 내용은 주문 관리를 참조하십시오.

주문 목록 ​

메인 테이블은 다음 열과 함께 기존 주문을 모두 표시합니다.

  • 프로젝트 이름: 주문과 연결된 프로젝트의 이름입니다.
  • 주문 번호: 주문의 고유 식별자입니다.
  • 주문 생성 시간: 주문이 생성된 날짜와 시간입니다.
  • 유형: 결제 유형입니다. 옵션은 ko-fi 주문 유형에 따라 기부, 구독입니다.
  • 주문 금액: 통화를 포함한 주문의 총액입니다.
  • 고객 이메일: 주문한 고객의 이메일 주소입니다.
  • 이메일 전송 상태: 주문 확인 이메일의 상태입니다. 옵션은 다음과 같습니다.
    • 0: NoMatch: 일치하는 결제 구성이 없습니다.

      참고

      이메일을 다시 보내려면 먼저 결제 페이지로 이동하여 필요한 결제 구성을 추가하십시오. 자세한 내용은 결제를 참조하십시오.

    • 1: 성공: 이메일이 성공적으로 전송되었습니다.
    • 2: 실패: 이메일 전송에 실패했습니다.
    • 3: MatchNoSend: 일치하는 결제 구성이 발견되었지만 자동 전송이 활성화되지 않았습니다.
  • 주문 추가 메시지: 결제 주문에 포함된 추가 메시지입니다.
  • 작업: 각 주문을 관리하는 옵션을 제공합니다.

사용 가능한 작업: ​

  • 상세 정보: 문서 아이콘을 클릭하여 주문과 연결된 원시 데이터를 볼 수 있습니다.
  • 이메일 재전송: 이메일이 자동으로 전송되지 않은 경우 이메일 재전송 아이콘을 클릭하여 이메일을 수동으로 재전송할 수 있습니다.

참고

이메일 내용과 제목은 결제 페이지에서 구성됩니다. 자세한 내용은 결제를 참조하십시오.

결론 ​

VpAdmin의 주문 페이지는 결제 주문을 관리하기 위한 완벽한 솔루션을 제공하여 모든 거래를 추적하고 모든 주문을 효과적으로 관리할 수 있도록 합니다. 주문 관리를 이메일 재전송 기능과 결합함으로써 이 모듈은 사용자에게 전문적이고 원활한 구매 경험을 유지하는 데 도움이 됩니다.

결제 확인 개요 ​

VpAdmin의 결제 확인 페이지는 웹사이트에서 시작된 결제 확인에 대한 포괄적인 보기를 제공합니다. 이 페이지를 통해 관리자는 결제 활동을 모니터링하고, 주문 상태를 관리하고, 사용자에게 응답하여 효율적인 거래 처리와 사용자 지원 강화를 용이하게 할 수 있습니다.

개요 ​

결제 확인 페이지에서는 다음을 수행할 수 있습니다.

  • 결제 확인 추적: 웹사이트에서 받은 모든 결제 확인 목록을 봅니다.
  • 결제 확인 필터링: ID, 프로젝트 이름, 결제 이름, 사용자 이메일, 상태 및 시간을 기준으로 결제 확인을 필터링합니다.
  • 결제 확인 세부 정보 보기: 연결된 결제 정보, 사용자 이메일, 상태 및 회신 내용을 포함하여 각 결제에 대한 자세한 정보를 봅니다.
  • 결제 상태 업데이트: 결제 상태를 수동으로 업데이트하고 사용자에게 회신 메시지를 설정합니다.
  • 일괄 결제 상태 업데이트: 한 번에 여러 결제 기록의 상태를 업데이트합니다.

결제 확인 목록 ​

기본 표에는 다음 열이 있는 모든 결제 확인 기록이 표시됩니다.

  • ID: 결제 확인 기록의 고유 식별자입니다.
  • 프로젝트 이름: 결제와 연결된 프로젝트의 이름입니다.
  • 결제 이름: 사용된 결제 구성의 이름입니다.
  • 이메일: 결제를 시작한 사용자의 이메일 주소입니다.
  • 상태: 결제의 현재 상태이며, 옵션은 다음과 같습니다.
    • 대기 중: 시스템에서 결제 확인을 받았으며 시스템에서 결제를 매칭하기를 기다리는 중입니다.
    • 성공: 결제가 완료되었고 kofi 주문과 일치합니다.
    • 실패: 결제 확인 매칭에 실패했습니다.
  • 시간: 결제 확인 기록 생성 시간입니다.
  • 작업: 각 결제 확인 기록을 관리할 수 있는 옵션을 제공합니다.

사용 가능한 작업 ​

  • 세부 정보 보기: "보기" 아이콘을 클릭하여 결제 확인의 세부 정보를 봅니다.
  • 세부 정보 대화 상자에는 다음 정보가 제공됩니다.
  • 상태 업데이트: "편집" 아이콘을 클릭하여 결제 확인 상태를 수정하고 회신 메시지를 추가합니다.
  • 이렇게 하면 상태를 업데이트하고 사용자에게 사용자 지정 회신을 제공할 수 있는 대화 상자가 열립니다.

주요 워크플로 ​

자동 결제 매칭 및 회신 ​

  1. 사용자 결제 시작: 사용자가 웹사이트의 지원 버튼을 사용하여 결제하면 VpAdmin은 "결제 확인" 페이지에 사용자의 결제 확인을 기록합니다.
  2. 웹후크 처리: Ko-fi의 결제가 성공적이고 결제 구성 페이지의 구성과 일치하는 경우 VpAdmin은 자동으로 주문과 결제를 매칭하려고 시도합니다.
    • 일치하는 결제 구성이 발견되면 VpAdmin은 자동으로 결제 상태를 '성공'으로 변경하고 자동 이메일 회신을 제공합니다.
    • 일치하는 결제 구성을 찾을 수 없는 경우 결제 기록은 "대기 중" 상태로 표시됩니다.
  3. 관리자 수동 조치 필요: 일치하는 주문이 없고 상태가 대기 중인 결제 확인 기록의 경우 관리자는 해당 결제 기록의 상태를 수동으로 변경하고 회신을 보내야 할 수 있습니다.

주문 관리 및 자동/수동 회신 원칙에 대한 자세한 내용은 주문 관리를 참조하십시오.

사용자 경험 ​

사용자는 사용자 센터 결제 섹션에서 상태와 관리자 회신을 볼 수 있습니다.

결론 ​

결제 확인 페이지는 VpAdmin 내에서 결제 확인을 관리하는 데 중요한 구성 요소입니다. VpAdmin은 결제 추적, 업데이트 및 응답 도구를 제공함으로써 관리자와 사용자 모두에게 투명성과 원활한 결제 경험을 보장합니다.
